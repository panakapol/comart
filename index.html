Instruction Format — 32 bits ต่อ 1 คำ (Word)

โครงสร้าง OP-Code, Register, Address
💡

โจทย์จะให้ว่า "มี 32 bits ต่อ 1 คำ" แล้วถามว่า OP-Code มีกี่ bit / Register มีกี่ bit / เกี่ยวกับ bit ต่างๆ

1.1) OP-Code มีกี่ bit?

ดูจาก Instruction Format — ส่วน OP-Code คือบิตแรกของ instruction
ถ้า format = Opcode (4 bit) | Mode (1 bit) | Address (11 bit) รวม = 16 bit
สำหรับ 32-bit word ต้องดูจาก diagram ที่ให้มาในโจทย์

1.2) Register มีกี่ bit?

Register field = บิตที่ใช้ระบุตัว register (เช่น R0–R7 ต้องการ 3 bit)

1.3) เกี่ยวกับ bit ต่างๆ (วาดรูปเบอะ)

โจทย์อาจให้วาด Instruction Format diagram แสดง field ต่างๆ เช่น:

| OP-Code | I (Indirect) | Register | Address |

เลข bit ในแต่ละ field รวมกันต้องได้ 32 bits เสมอ

1.4) จำนวน / 1.5) อื่นๆ

อาจถามจำนวน instruction สูงสุดที่เป็นไปได้ = 2n (n = จำนวน bit ของ OP-Code)

OP-Code

ระบุประเภทของคำสั่ง เช่น ADD, LDA, STA
ถ้า n bit → มีได้ 2n คำสั่ง

Address Field

ระบุตำแหน่งหน่วยความจำที่จะเข้าถึง
เช่น 12 bit → address space = 212 = 4096

Register & AR (Address Register)

ตัวโหนดเกี่ยวกับ Register ในระบบ Common Bus

AR คือ Address Register — ใช้เก็บ ที่อยู่ (Address) ของหน่วยความจำที่จะอ่าน/เขียน

Registerชื่อเต็มหน้าที่
ARAddress Registerเก็บ address ของ memory ที่จะเข้าถึง
PCProgram Counterเก็บ address ของ instruction ถัดไป
IRInstruction Registerเก็บ instruction ที่กำลัง execute
DRData Registerเก็บข้อมูลที่อ่านมาจาก memory
ACAccumulatorเก็บผลลัพธ์ของการคำนวณ ALU
TRTemporary Registerเก็บข้อมูลชั่วคราว
🔑

AR ดูที่ติ๊กไว้ — AR มี input จาก: PC (ตอน Fetch), DR[0-11] (ตอน Indirect), และ IR[0-11] (ตอน Decode)
การที่ AR จะโหลดค่าได้ต้องมีสัญญาณ L (Load) และ clock อยู่ในสภาวะที่ถูกต้อง

โจทย์อาจถาม: AR โหลดค่าเมื่อไหร่?

ดูจาก Microoperations:
• R₀T₁: AR ← PC (ตอน Fetch)
• R₀T₂: AR ← 0, TR ← PC (ตอน Interrupt)
• D₅T₃ (Indirect): AR ← M[AR] (ถ้า I=1)
สัญญาณที่ทำให้ AR load = R₀T₁ OR D₅T₃ OR T₄ (...)

Microoperations — LDA / ADD

อธิบายว่าเกิดอะไรขึ้นในแต่ละ cycle

โจทย์จะให้ sequence ของ microoperation แล้วถามว่า "อธิบายว่าเกิดอะไรขึ้น"

Fetch Cycle (ทุก Instruction เริ่มที่นี่)
T₀: AR ← PC
T₁: IR ← M[AR], PC ← PC + 1
T₂: AR ← IR[0-11], I ← IR[15], Decode IR[12-14]
LDA (Load to Accumulator) — D₂
D₂T₄: DR ← M[AR]
D₂T₅: AC ← DR, SC ← 0

คือ: อ่านค่าจาก memory มาเก็บที่ DR แล้วโอนไป AC

ADD — D₁
D₁T₄: DR ← M[AR]
D₁T₅: AC ← AC + DR, E ← Cout, SC ← 0

คือ: อ่านค่าจาก memory → บวกกับ AC → ผลลัพธ์เก็บที่ AC, carry เก็บที่ E

⚠️

SC ← 0 หมายถึง Sequence Counter รีเซ็ต = จบ cycle นั้น กลับไปเริ่ม Fetch ใหม่

Prefix Expression → สมการทั่วไป

3-Address / แปลง Prefix เป็น Infix
วิธีแปลง Prefix → Infix
  • อ่าน Prefix จากซ้ายไปขวา
  • พบ Operand → push ลง stack
  • พบ Operator → pop 2 ค่าจาก stack, สร้าง (op1 operator op2), push กลับ
  • ทำซ้ำจนหมด → ผลใน stack คือคำตอบ
ตัวอย่าง: + A * B C → Infix
Prefix: + A * B C
อ่านจากขวา: C, B, * → (B * C)
ต่อ: A, + → (A + (B * C))
Infix: A + B * C
โจทย์จากเล่ม: ((A + B) + C) XOR ...

ต้องแปลงจาก Prefix กลับเป็น Infix พร้อมใส่วงเล็บให้ถูกต้องตาม precedence

Prefix (Polish)

Operator นำหน้า Operand
เช่น + A B = A + B

Postfix (Reverse Polish)

Operator ตามหลัง Operand
เช่น A B + = A + B

6

Pipeline — D[i] = A[i]*B[i] + A[i]*C[i]

Pipelined vs Non-Pipelined, Speedup

โจทย์ให้ operation: D[i] = A[i]*B[i] + A[i]*C[i] สำหรับ i = 1,2,3,...,7 (i=0 ด้วย)

6.1) วาดออกมาประมาณนี้ (Pipeline Diagram)

Pipeline 3 Segment:

Segment 1: R1 ← A[i], R2 ← B[i] (Load A, B)
Segment 2: R3 ← R1 * R2, R4 ← C[i] (Multiply & Load C)
Segment 3: R5 ← R3 + R4 (Add)
6.2) ความวิธีลดเวลา / 6.1) มาก

Pipeline ช่วยให้แต่ละ segment ทำงานพร้อมกัน (Concurrent) แทนที่จะทำทีละขั้น

6.3) Speedup Formula — สำคัญมาก ⭐
Non-Pipelined: τ = n × t_n
Pipelined (k stages): τ_p = (k + n - 1) × t_p

Speedup: S_k = (n × t_n) / ((k + n - 1) × t_p)

เมื่อ n → ∞: S_k → t_n / t_p = k (ถ้า t_n = k × t_p)

ตัวแปร: n = จำนวน tasks k = จำนวน stages t_p = clock cycle ของ pipeline

📌

จาก 6.2 และ 6.1 เทียบกัน — 6.2 ดีกว่า เพราะไม่ต้องเปรียบเทียบกับ 6.1 เหมือนกัน (Pipeline เร็วกว่า Non-Pipeline)

Assembly Language

เขียน Assembly จาก Pseudocode + แปลงเป็นฐาน 16
โจทย์ (Pseudocode ที่ให้มา)
i = -5, sum = 0
while (i ≤ 0) {
  if (i == -2 || i == -4)
    sum = sum + (-i - i) // บวกสองเท่า
  else
    sum = sum + (-i)
  i = i + 1
}
output = sum
7.1) เขียนเป็น Assembly (ตัวอย่าง Assembly ที่ให้มา)
; ตัวอย่าง Assembly โครงสร้าง
    SUM = 0
    C = -5
LOOP: while C <= 0
    if C ∈ {9,-2,54} ; ตามที่โจทย์กำหนด
      SUM = SUM - C
    end if
    C = C + 1
END: end while
7.2) แปลงจาก Assembly เป็นฐาน 16 (Hex)

แปลงค่าต่างๆ ใน Assembly เป็นเลขฐาน 16:

ค่า Decimalค่า Hex (ฐาน 16)ค่า Binary
00x000000 0000
-50xFB (2's complement)1111 1011
-20xFE1111 1110
-40xFC1111 1100

วิธีแปลง negative → Hex: หา 2's complement แล้วแปลงเป็น Hex

💡

โจทย์ถามเขียนโปรแกรม + Assembly + แปลงเลขฐาน — มีเขียน loop แล้วก็ if แปลงเลขฐาน
ข้อสอบมีประมาณ 7 ข้อ แล้วอาจจะพูดแต่ละครั้งไม่เหมือนเดิม อย่าถาม!

Instruction Format (32-bit Word)

ข้อ 1 ในข้อสอบ
โจทย์: ระบบมี 32 bits ต่อ 1 คำ (word) — หา จำนวน bits ของ OP-code, Register, Address, และอื่นๆ ตามที่โจทย์กำหนด
โครงสร้าง Instruction 32-bit

Instruction word ขนาด 32 bits แบ่งออกเป็น field ต่างๆ ขึ้นกับ format:

Fieldขนาด (bits)หน้าที่
OP-Codeขึ้นกับโจทย์ (เช่น 4–8 bits)ระบุประเภทคำสั่ง
Register (Reg)log₂(จำนวน register)ชี้ register ที่ใช้
Address / Operandที่เหลือ = 32 − OP − Regชี้ตำแหน่ง memory หรือ immediate
วิธีคำนวณ (ตัวอย่าง)
ตัวอย่าง 1:
Word = 32 bits, OP-code = 6 bits, Registers = 2 fields × 5 bits
→ Address = 32 − 6 − 5 − 5 = 16 bits
→ Memory = 2¹⁶ = 65,536 locations
ตัวอย่าง 2 (3-address format):
ADD R1, R2, R3
OP(4) + R1(4) + R2(4) + R3(4) = 16 bits → เหลือ 16 bits unused
จำนวน Instructions สูงสุด = 2OP-bits
จำนวน Registers สูงสุด = 2Reg-bits
ขนาด Memory สูงสุด = 2Address-bits words
ประเภท Instruction Format
3-Address
ADD R1, R2, R3 ; R1 ← R2 + R3 ; ใช้ bits มาก แต่ flexible
2-Address
ADD R1, R2 ; R1 ← R1 + R2 ; ประหยัด bits
1-Address (Accumulator)
ADD M[X] ; AC ← AC + M[X] ; Basic Computer ใช้แบบนี้
📝 ข้อสอบ 1.1–1.5 (จากลายมือ)
ถ้ามี 32 bits ต่อ 1 word:
1.1) จะมี OP-Code กี่ bit?    1.2) Register มีกี่ bit?    1.3) เกี่ยวกับ bit ตามรูป    1.4) กี่ km (ขนาด memory?)    1.5) กี่ km
✅ แนวตอบ
ต้องดูรูปโจทย์ประกอบ — ใช้สูตร: OP + Reg + Address = 32 bits เสมอ
จำนวน OP-code bits = กำหนดจากโจทย์ (ดูรูป Instruction format)
Register bits = log₂(n) เมื่อ n = จำนวน register
Address bits = 32 − OP − Reg
Memory size = 2^(Address bits) × Word size bytes

✅ แนวตอบ

AR เป็น Register ที่สำคัญที่สุดเพราะทุก memory access ต้องผ่าน AR

05

Pipelining — การทำงานแบบ Pipeline

ข้อ 8 เพิ่มเติม
Pipelining คืออะไร

Pipeline คือการแบ่ง task ออกเป็น sub-task หลายขั้น แต่ละขั้นทำงานพร้อมกัน เหมือนสายการผลิต

สูตร Pipeline Performance
τ_p = (k + n - 1) × t_p
k = จำนวน stage, n = จำนวน tasks, t_p = clock cycle ของ pipeline
S_k = n×t_n / (k+n-1)×t_p
Speedup เทียบกับ non-pipelined
lim(n→∞) S_k = t_n/t_p = k (ถ้า t_n = k×t_p)
ตัวอย่าง: k=3 stages, n=7 tasks, t_p=1ns
τ_p = (3+7-1)×1 = 9ns
t_n = 3ns (non-pipeline: 3×7=21ns)
Speedup = 21/9 ≈ 2.33
Pipeline Hazards ข้อ 8
Structural Hazard: hardware resource ขัดแย้ง (ใช้ unit เดียวกันพร้อมกัน)
Data Hazard (RAW): คำสั่งถัดไปต้องการผลลัพธ์ที่ยังไม่เสร็จ
R1←R2+R3 แล้ว R4←R1+1
Control Hazard: Branch instruction เปลี่ยน PC → fetch ผิด instruction
แก้ปัญหา: Pipeline Interlock (Stall), Forwarding/Bypassing, Branch Prediction, Detect Hazards Stall

Vector Pipeline — D[i] = A[i]*B[i] + A[i]*C[i]

ข้อ 6 ในข้อสอบ
วงจร 3-Segment Pipeline

สำหรับ i = 1,2,3,...,7: D[i] = A[i]*B[i] + A[i]*C[i]

6.1) จาก Segment ข้างต้น: ใช้ 3 segments, แต่ละ i ต้องการ 3 clock cycles
6.2) การลดเวลา: Pipeline ให้ทำ i=1,2,...,7 ทับซ้อนกัน → เวลารวม = (3+7-1) = 9 cycles
6.3) Speedup: ไม่ใช้ pipeline = 7×3 = 21 cycles, ใช้ pipeline = 9 cycles → Speedup ≈ 2.33

Software Approaches ▼
Read-Only Cacheable:
- Cache เก็บได้เฉพาะข้อมูลที่ read-only

- เก็บ status ของแต่ละ memory block: RO / RW
- RO: ทุก cache มี copy ได้
- RW: มีได้แค่ 1 cache เท่านั้น
Hardware Approach: Snoopy Cache ▼
Snoopy Cache Controller:
- Cache controller monitor bus requests ทั้งหมด
- ทุก cache "snoop" (แอบดู) bus operations
- ถ้า P1 เขียนค่า X → memory อัปเดต (write-through)
- Controller อื่นตรวจว่ามี copy ไหม → ถ้ามี mark = INVALID
- การอ่าน X ใหม่ → cache miss → ดึงจาก memory (อัปเดตแล้ว)
Semaphore — Mutual Exclusion
Semaphore คือ binary variable (0 หรือ 1)
1 = มี processor กำลังใช้ critical section
0 = พร้อมให้ใช้งาน
/ ตรวจและตั้ง Semaphore (แบบ indivisible) / R ← M[SEM] / Test semaphore / M[SEM] ← 1 / Set semaphore / / ทำพร้อมกันแบบ locked / if R = 1: / อีก processor ใช้อยู่ → รอ / if R = 0: / ว่าง → เข้า critical section / / ... ทำงาน ... / M[SEM] ← 0 / clear เมื่อเสร็จ /

Instruction Format (32-bit per word)

Instruction Format — OP-Code, Registers, Addressing
OP-Code bits Register bits Address bits Word size

โจทย์กำหนด: มี 32 bits ต่อ 1 คำ (word) แล้วถามเกี่ยวกับ OP-Code, Register, และ bit ต่างๆ

🔲 โครงสร้าง 32-bit Instruction Format
1.1) OP-Code มีกี่ bit → มีคำสั่งได้กี่คำสั่ง
-- ถ้า OP-Code = 8 bits -- จำนวนคำสั่งสูงสุด = 2^8 = 256 คำสั่ง -- สูตรทั่วไป -- OP-Code = n bits → คำสั่งได้ = 2^n
1.2) Register มีกี่ bit → มีกี่ register
-- ถ้า Register field = 6 bits -- จำนวน register สูงสุด = 2^6 = 64 registers -- สูตรทั่วไป: register field = r bits -- จำนวน registers = 2^r
1.3) ตามเกี่ยวกับ bit (วาดรูปของเอง)

ดูรูป Instruction Format ด้านบน — ต้องจำว่าแต่ละ field กว้างกี่ bit และอยู่ที่ตำแหน่งใด

1.4) จำนวน Address bits
Address bits = 32 - OP-Code bits - Register bits เช่น: 32 - 8 - 12 = 12 bits → addressable memory = 2^12 = 4096 locations
🔑 จุดสำคัญ

OP-Code กำหนดจำนวน Instruction ที่รองรับได้
Register field กำหนดจำนวน Register ในระบบ
Address field กำหนดขนาด Memory ที่เข้าถึงได้

📐 สูตรที่ใช้บ่อย

n-bit field → 2ⁿ ค่าที่เป็นไปได้
Word = 32 bit → OP + Reg + Addr = 32
Memory size = 2^(address bits) words

💡

โจทย์มักให้ total bits = 32 แล้วถามให้หา: กี่ bits เป็น OP-Code, กี่ bits เป็น register, กี่ register, กี่ instructions

Operator นำหน้า

Operator ตาม
A B + C *

วิธีแปลง Prefix → Infix ด้วย Stack
อ่านจากขวาไปซ้าย: 1. พบ Operand → PUSH ลง stack 2. พบ Operator → POP 2 operand, สร้าง (op1 OP op2), PUSH กลับ 3. ทำซ้ำจนหมด → ค่าใน stack = คำตอบ ตัวอย่าง: แปลง * + A B C อ่านจากขวา: C → push[C] B → push[C,B] A → push[C,B,A] + → pop A,B → (A+B) → push[C,(A+B)] * → pop (A+B),C → ((A+B)*C)
โจทย์: แปลง ((A + B) + C) XOR ... เป็น Prefix
วิธีแปลง Infix → Prefix: 1. ใส่วงเล็บ expression ตาม precedence ให้ครบก่อน 2. ย้าย operator มาหน้า operand ของมัน 3. ลบวงเล็บออก ตัวอย่าง: (A + B) * C → (* (+ A B) C) → * + A B C
3-Address Instruction
Format: OP dest, src1, src2 ADD R1, R2, R3 → R1 = R2 + R3 MUL R4, R1, R5 → R4 = R1 * R5 ข้อดี: ไม่ต้องเขียน operand ซ้ำ, code สั้น ข้อเสีย: instruction ยาว (ต้องระบุ 3 addresses)

6.3) Speedup Formula — ⭐ ออกสอบแน่นอน
-- Non-Pipelined -- τ = n × tₙ (n tasks × เวลาต่อ task) -- Pipelined (k stages) -- τₚ = (k + n - 1) × tₚ (clock cycles × clock period) -- Speedup -- Sₖ = (n × tₙ) / ((k + n - 1) × tₚ) -- เมื่อ n → ∞ -- lim Sₖ = tₙ / tₚ = k (ถ้า tₙ = k × tₚ) n→∞
ตัวอย่างโจทย์คำนวณ
กำหนด: n=7 tasks, k=3 stages, tₙ=30ns, tₚ=10ns Non-Pipeline: τ = 7 × 30 = 210 ns Pipeline: τₚ = (3 + 7 - 1) × 10 = 9 × 10 = 90 ns Speedup: Sₖ = 210 / 90 = 2.33 Max speedup (n→∞) = tₙ/tₚ = 30/10 = 3

Amdahl's Law — ข้อจำกัดของ Speedup
f = fraction ของส่วนที่ต้องทำ Sequential p = จำนวน processors Speedup S ≤ 1 / (f + (1-f)/p) ตัวอย่าง: f = 0.1 (10% sequential), p = 10 processors S ≤ 1 / (0.1 + 0.9/10) = 1 / 0.19 ≈ 5.26 เมื่อ p → ∞: S_max = 1/f ถ้า f=0.1: S_max = 1/0.1 = 10 (ไม่ว่าจะเพิ่ม CPU เท่าไหร่)
🌐 Interconnection Networks
Cache Coherence — ปัญหาและวิธีแก้
ปัญหา: P1 เขียน X=120 ใน cache ของตัวเอง แต่ P2, P3 ยังเห็น X=52 (ค่าเก่า) ใน cache ของตัวเอง → Cache Incoherency! วิธีแก้: 1. Write Through: เขียน cache → เขียน memory ทันที แต่ cache ของ P2,P3 ยังเก่าอยู่ 2. Snoopy Cache: Cache controllers monitor bus เมื่อ P1 เขียน → P2,P3 invalidate copy ของตัวเอง 3. Software (CGT): Centralized Global Table RO blocks: หลาย cache คัดลอกได้ RW blocks: มีแค่ 1 cache ที่มีได้
Semaphore — Mutual Exclusion
Semaphore = binary variable (0 หรือ 1) 1 = Critical Section กำลังถูกใช้ (ล็อก) 0 = พร้อมให้ใช้ Test & Set (Indivisible operation): R ← M[SEM] ; ดูค่า semaphore M[SEM] ← 1 ; ล็อกไว้ if R = 1 → อีก processor ใช้อยู่ → รอ if R = 0 → เข้า critical section ได้ → ตั้ง SEM=1 เสร็จแล้ว → M[SEM] ← 0 (ปลดล็อก)
